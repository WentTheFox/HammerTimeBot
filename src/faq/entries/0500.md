# Can you create a timestamp that always points to a specific time of day every day/week/month?
Due to the "limitations" of the timestamp syntax it's only possible to create a timestamp for a specific point in time by design. "5pm Friday" has a lot of implications, for example in case the next Friday is in a different timezone due to daylight savings (DST). It's difficult to reliably translate a recurring point in time like that without avoiding edge cases, but if you need this functionality either manually update the timestamp each week to the next occasion, create a bot that does this for you. This will not be part of the HammerTime bot.
